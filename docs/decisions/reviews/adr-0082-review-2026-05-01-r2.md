## 4. Observations

### O2.1 — Non-goals vs. implementation requirement 8 (OBSERVATION, no action required)

The Non-goals section states: "Defining which specific CloudWatch alarms to create. The ADR defines alarm class taxonomy and delivery architecture; specific alarms are Terraform-level implementation."

Implementation requirement 8 explicitly specifies the ECS `RunningTaskCount` alarm. This is technically in tension with the non-goal, but the tension is acceptable: the ECS task failure alarm is architecturally significant because it directly detects the dead-man-switch scenario — the core problem this ADR solves. It is not a general alarm specification; it is a structural requirement of the fallback architecture.

**Recommendation:** No revision required. The non-goal refers to general alarm selection (which specific CPU thresholds, which log patterns, etc.). The `RunningTaskCount` alarm is a structural element of the fallback architecture, not a general alarm specification.
